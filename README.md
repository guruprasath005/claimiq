# Stobaeus Docx — Insurance Claim RCM Tool

AI-powered Revenue Cycle Management tool for Indian hospitals. Upload case sheet images, extract all clinical and insurance data, cross-reference against indexed insurer policy trees, and get a claim verdict with actionable checklists — all running on a local Gemma 4 12B model.

---

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  Browser  →  React + Vite + Tailwind  (port 5173)          │
└────────────────────────────┬────────────────────────────────┘
                             │ REST / JSON
┌────────────────────────────▼────────────────────────────────┐
│  FastAPI backend  (port 8000)                               │
│                                                             │
│  /upload    → OCR (multi-page)  ─────────┐                 │
│  /evaluate  → PageIndex search            │                 │
│             → ICD-10 matching             ▼                 │
│             → Claim evaluation   local vllm  (Gemma 4 12B) │
│  /cases     → PostgreSQL CRUD            │                 │
│  /schemes   → insurer list               │ fallback        │
│                                          ▼                 │
│                                  OpenAI GPT-4o-mini        │
└─────────────────────────────────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│  PostgreSQL  (cases, ICD codes, missing items, sections)    │
└─────────────────────────────────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│  indexer/tree_index/  — PageIndex JSON trees per insurer    │
│  (built once with index.sh / start.bat --index)             │
└─────────────────────────────────────────────────────────────┘
```

## Features

- **Multi-page OCR** — upload 1–N case sheet images; each page is processed in parallel by Gemma, then consolidated into one structured record
- **Comprehensive extraction** — 12 sections: patient, admission, doctors, vitals, clinical, investigations, diagnosis, surgery, procedures, medications, insurance, facility
- **Editable review** — doctor reviews and corrects every field before evaluation
- **PageIndex tree search** — 16+ insurers indexed from policy PDFs and web clippings; Gemma navigates the tree to find relevant coverage/exclusion sections
- **ICD-10 coding** — codes generated by the LLM based on diagnoses + policy context
- **Claim verdict** — APPROVABLE / PARTIAL / REJECTED / UNKNOWN with full reasoning
- **Three-checklist output** — missing/incomplete fields · documents to collect · documents to generate
- **Token usage report** — per-stage breakdown (OCR → index search → ICD match → evaluation)
- **Gemma → GPT-4o-mini fallback** — if the local vllm endpoint is unreachable, calls automatically retry against OpenAI

## Quick Start

### Linux / macOS

```bash
# 1. Clone
git clone https://github.com/guruprasath005/stobaeus-docx.git
cd stobaeus-docx

# 2. Configure
cp .env.example .env
# Edit .env — set OPENAI_BASE_URL, DATABASE_URL, FALLBACK_API_KEY

# 3. Database
psql $DATABASE_URL -f db/schema.sql

# 4. Python deps
python3 -m venv .venv && source .venv/bin/activate
pip install -r backend/requirements.txt

# 5. Frontend deps
cd web && npm install && cd ..

# 6. Run
./start.sh
```

Open **http://localhost:5173**

### Windows

```bat
:: 1. Clone and enter directory
git clone https://github.com/guruprasath005/stobaeus-docx.git
cd stobaeus-docx

:: 2. Configure
copy .env.example .env
:: Edit .env with Notepad

:: 3. First-time setup (installs deps + applies DB schema)
start.bat --setup

:: 4. Run
start.bat
```

## Windows `start.bat` Modes

| Command | What it does |
|---|---|
| `start.bat` | Start backend + frontend, open browser |
| `start.bat --setup` | Create `.venv`, install deps, apply DB schema |
| `start.bat --index` | Build PageIndex trees from `InsuranceData/` |

## Configuration (`.env`)

| Variable | Default | Description |
|---|---|---|
| `OPENAI_BASE_URL` | `http://10.10.116.160:11632/v1` | vllm endpoint URL |
| `LLM_MODEL` | `gemma-4-12b-it` | Primary model name |
| `FALLBACK_API_KEY` | — | OpenAI API key for GPT-4o-mini fallback |
| `FALLBACK_MODEL` | `gpt-4o-mini` | Fallback model name |
| `DATABASE_URL` | `postgresql://localhost:5432/stobaeus` | PostgreSQL connection string |
| `INDEX_MODEL_URL` | same as `OPENAI_BASE_URL` | vllm URL used by the indexer |
| `INDEX_MODEL` | `gemma-4-12b-it` | Model used to build PageIndex trees |
| `CORS_ORIGINS` | `["http://localhost:5173"]` | Frontend origins allowed by CORS |
| `MAX_UPLOAD_MB` | `20` | Per-image upload size limit |

## Building the Insurance Index

The indexer reads policy PDFs and web clippings from `InsuranceData/` and builds JSON PageIndex trees in `indexer/tree_index/`.

```bash
# Linux / macOS
./index.sh

# Windows
start.bat --index
```

The index only needs to be rebuilt when policy documents change. Each insurer gets up to three tree types: `policy_docs`, `claims`, `policies_services`.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `POST` | `/upload` | Upload 1–N images, run OCR, return structured extraction |
| `POST` | `/evaluate` | Run full pipeline: index search → ICD → verdict |
| `GET` | `/schemes` | List indexed insurers |
| `GET` | `/cases` | List all cases |
| `GET` | `/cases/{id}` | Case detail |
| `PATCH` | `/cases/{id}` | Update verdict |
| `GET` | `/reports/summary` | Dashboard metrics |
| `GET` | `/settings` | Current model config |
| `PATCH` | `/settings` | Update model config |
| `GET` | `/health` | Health check |

**AI** — Gemma 4 12B via local vllm (OpenAI-compatible endpoint), GPT-4o-mini fallback
